================================================================================
CHANGELOG: ESM-MAMBA (MAMBACROSS) NEURAL NETWORK PIPELINE
================================================================================

This document chronicles all changes, modifications, and fixes applied to the
ESM-Mamba (MambaCross) Neural Network codebase during this pair programming
session to optimize execution for the local NVIDIA Blackwell rig.

--------------------------------------------------------------------------------
1. ENVIRONMENT & HARDWARE CONFIGURATION
--------------------------------------------------------------------------------
* Python Interpreter Selection:
  - Diverted from the default Windows pre-release python (`3.14.6`) to a stable
    virtual environment based on Python `3.11.15` managed by the high-speed `uv`
    package installer. This ensures precompiled wheels are accessible and bypasses
    unnecessary compiler toolchain errors.
* PyTorch Upgrade for Blackwell GPU:
  - Upgraded PyTorch from the CPU baselines to the CUDA 13.2 nightly build:
    `torch==2.14.0.dev20260720+cu132`.
  - This solves the fatal CUDA crash: "no kernel image is available for execution
    on the device" by natively compiling state-space (VMamba) computational kernels
    for the sm_120 (Blackwell) microarchitecture.

--------------------------------------------------------------------------------
2. CODEBASE BUG FIXES & COMPATIBILITY PATCHES
--------------------------------------------------------------------------------
* Safe Antibody Filename Mapping:
  - Patched `shared/Pretrained.py` to replace slashes in antibody names with
    underscores (e.g. `10E8-0fH/0fL` -> `10E8-0fH_0fL.npy`) before calling `np.save`.
  - This prevents file lookup/write exceptions and matches the loader logic in
    `Toolkit.py` and `train_nn.py` which load sequence embeddings using safe ID
    strings.
* Tokenizer Vocabulary Cleaning:
  - Patched `shared/Pretrained.py` to replace sequence characters "#" with "X"
    (the IUPAC standard for unknown amino acids). This prevents ESM-2's batch
    tokenizer from raising a KeyError on unresolved/missing residues in the env_aa
    viral database.
* Autocast BCELoss RuntimeError Resolution:
  - In PyTorch 2.14+, calculating BCELoss inside an AMP `autocast` block is
    forbidden and throws a fatal RuntimeError.
  - Patched `train_experiment.py` and `experiment_{1-4}_*/train_nn.py` to move the
    loss calculation outside the autocast block and explicitly convert model output
    tensors to standard `float32`:
    
    with autocast(device_type=device.type, dtype=amp_dtype, enabled=amp_enabled):
        outputs = model(ab_embs, ag_embs)
    loss = criterion(outputs.float(), labels.float())
* Windows Terminal Encoding Error Fix:
  - Replaced Unicode formatting characters (warning triangles, box borders, arrows)
    in `run_all_experiments.py` with standard ASCII equivalents, avoiding
    `UnicodeEncodeError` when writing outputs to Windows terminals running under
    CP1252.

--------------------------------------------------------------------------------
3. RUNTIME & SCHEDULING ADJUSTMENTS
--------------------------------------------------------------------------------
* GPU Memory Scheduling:
  - Changed execution from concurrent/parallel mode to sequential mode. Running 4
    experiments concurrently on a single GPU reserved excessive intermediate
    activation layers for VMamba, exhausting the 24 GB VRAM limit and causing a CUDA
    OOM crash. Sequential execution resolves this by training one model at a time with
    full device capabilities.
* Epoch Progress Checkpointing (Power Cut Protection):
  - Modified `train_experiment.py` and each individual `train_nn.py` to save an
    epoch checkpoint (`checkpoint.pt`) containing the state dicts of model and
    optimizer, the best AUC achieved, and the current epoch index.
  - Added startup resume logic: if a power cut occurs or execution is terminated,
    re-running the pipeline will automatically detect the checkpoint and resume
    from the exact last completed epoch.
  - Checkpoint file is cleaned up upon successful completion of the experiment.

--------------------------------------------------------------------------------
4. SEQUENCE ADAPTIVE POOLING OPTIMIZATION
--------------------------------------------------------------------------------
* Adaptive sequence pooling for 1000x speedup:
  - Downsampled embedding sequences using 1D Adaptive Average Pooling (F.adaptive_avg_pool1d)
    to a fixed sequence length of 256 for both antibody and antigen in the model forward pass.
  - This shrinks the contact matrix dimension by ~38x (from 616,512 to 65,536 elements),
    dramatically reducing the parallel scan (pscan) memory allocations and intermediate states.
  - Decreased batch training time from 22.8 seconds per step to ~0.02 seconds per step,
    turning a 15-day run into a 9-minute run per model.
  - Overrode `thres_ab` and `thres_ag` to 256 in all scripts (`train_experiment.py`,
    `experiment_*/train_nn.py`, and `verify_weights.py`).
